Design Rationale

This visualization explores: How could climate change reshape tourism comfort across different US regions by 2100? We built a Tourism Climate Suitability Index (TCSI) from CMIP6 climate model data, combining temperature and precipitation variables into a single 0–1 score per region per year, then compared it across three future emissions scenarios.

Why interaction is essential here

A static map freezes time; a static line chart loses geography. The linked choropleth map + time-series chart is the only layout that answers both "where?" and "when?" simultaneously. Most critically, the season selector reveals patterns no static plot can show — the South's summer TCSI collapses under SSP5-8.5 while the Northwest Mountain region's winter score stays resilient. This tradeoff is invisible in an annual-only view and is only discoverable through interaction.

Visual Encodings

Color uses a binned 6-level RdYlGn scale so readers instantly classify regions as poor/fair/good without reading numbers. Position on the line chart encodes TCSI over time for trend detection. Scenario tabs use consistent color (blue=low, amber=mid, red=high emissions) to reinforce severity ordering.

Data Transformations

Monthly CMIP6 temperature (tas) and precipitation (pr) are aggregated to annual means, then spatially averaged over four US regions using cosine-latitude weighting to correct for grid cell size distortion. TCSI applies a trapezoidal comfort function with region-specific ideal temperature ranges and precipitation penalties.

Alternatives Considered

We considered small-multiples (one map per scenario) but rejected it because it prevents direct year-by-year comparison. We also considered a scatter plot of TCSI vs. year but found geographic context (the map) essential — knowing where is as important as knowing when for a tourism framing.

Color Design

The RdYlGn diverging palette maps intuitively: red = poor tourism climate, green = ideal. We chose 6 discrete bins over a continuous gradient because binning makes regional comparisons faster and avoids implying false precision in model output. Region line colors are distinct and colorblind-accessible.
